Part A: OS Security & Malware

Weight: 1 Unit · Estimated time: ~1.5 hours

💻

The OS mediates every interaction between user, applications, and hardware, making it the prime target for attackers. Malware is any software written with malicious intent, varying by delivery method, persistence, and payload. Explore the families below.

Key Malware Families Map

Step A-1: Audit OS Baseline

Understand what "normal" looks like.

# Action items on your local VM/Machine:

1. List processes (ps aux / Get-Process)

2. List autostart services (systemctl / Get-Service)

3. List active network connections (netstat)

4. Check scheduled tasks (crontab / Get-ScheduledTask)

Deliverable A-1

Annotated table (≤10 rows) of notable entries. For each, write one sentence explaining its function and potential concern.

Step A-2: Static Malware Analysis

Examine a safe sample without execution.

  • Obtain a benign sample from MalwareBazaar or EICAR.
  • Compute the hash (sha256sum / Get-FileHash).
  • Submit hash to VirusTotal.
  • Examine strings for URLs, paths, keys.
  • Check file headers.

Deliverable A-2

One paragraph describing hash, VirusTotal score, interesting strings, and inferred behaviour. Attach one VT screenshot.

Step B-1: Network Reconnaissance

Run against localhost/isolated VM only.

nmap -sV -O -T4 127.0.0.1

Capture live HTTP traffic with Wireshark (optional/advanced).

Check firewall rules (iptables or netsh).

Deliverable B-1

Table of open ports, detected services, versions, and a 1-sentence risk commentary per port. Add Wireshark screenshot if applicable.

Step B-2: Web App Vulnerability Analysis

Test on DVWA, WebGoat, or Juice Shop only.

  • Review OWASP Top 10 and pick two categories.
  • Demonstrate SQL Injection (e.g., ' OR '1'='1).
  • Demonstrate Reflected XSS.
  • Review HTTP Security Headers (CSP, HSTS, X-Frame-Options) via Browser DevTools.

Deliverable B-2

(a) Paragraph on SQLi payload/result, (b) XSS proof screenshot, (c) Table of HTTP headers present/absent with risk explanation.

Step B-3: Propose Mitigations

Deliverable B-3

For each vulnerability from B-2, write 4-6 sentences detailing: Root Cause → Technical Fix → Supported Standard.
