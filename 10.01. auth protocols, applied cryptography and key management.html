The Power of MFA

Multi-Factor Authentication combines ≥2 factors: Something you KNOW (password), HAVE (phone/YubiKey), or ARE (fingerprint).

Interactive: How Password Storage Actually Works

Servers NEVER store your real password. They generate a random "salt" and combine it with your password using a slow Key Derivation Function (KDF) like bcrypt or Argon2 to output a hash. Try it below to see why identical passwords yield different database entries.

Lesson 2: Applied Cryptography

How do we hide data in plain sight?

Cryptography is the mathematical engine of security. Here, you will explore the CIA Triad, compare Symmetric and Asymmetric encryption techniques, and review the current industry standards for cryptographic algorithms to understand what is safe to use in 2026.

The CIA Triad

Every secure communication aims for these three goals. Modern AEAD modes (like AES-GCM) deliver all three in a single operation.

Confidentiality No one else reads it
Integrity No altering undetected
Authenticity You know who sent it

The Two Families

Feature Symmetric Asymmetric
Keys 1 shared secret 2 (Public + Private)
Examples AES, ChaCha20 RSA, ECDSA, Ed25519
Speed Very Fast 100-1000x Slower
Use Case Bulk data encryption Key exchange, signatures
*Every real system is a hybrid: asymmetric negotiates the key, symmetric protects the bulk data.

Algorithm Status · 2026

Never roll your own crypto. Use audited libraries and approved algorithms.

⚠️ Why Implementation Matters

Historic failures like WEP Wi-Fi, Sony PS3 ECDSA breach, Heartbleed, and ROCA all share one trait: the underlying math was fine; the implementation was flawed.

Golden Rule: Reusing a nonce under the same symmetric key leads to catastrophic failure.

Perfect Forward Secrecy (PFS)

Using Ephemeral DH (ECDHE), a new key is generated per session. If an attacker records traffic today and steals the server's private key five years from now, they cannot decrypt the old traffic because the session keys were never stored.

Convergence: The TLS 1.3 Handshake

How the three lessons fit together in under 100 milliseconds.

Every time you open an HTTPS website, an intricate choreography occurs. This final section demonstrates how Authentication, Cryptography, and Key Management combine to form a secure channel, executed ~5 billion times per second worldwide.

The End-to-End Encrypted Channel

1

Authentication

The server presents its Certificate (signed by a trusted Root CA). Your browser verifies the digital signature (Lesson 1 & 2) to prove the server's identity.

2

Key Exchange

Client and server use ephemeral Elliptic-Curve Diffie-Hellman (ECDHE) to derive a shared session key without ever sending it across the wire (Lesson 3).

3

Bulk Encryption

The agreed-upon session key is fed into a symmetric AEAD algorithm like AES-GCM or ChaCha20-Poly1305. All subsequent application data is encrypted and authenticated (Lesson 2).
